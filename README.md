# quotes-generator# Генератор на позитивни цитати (quotes-generator)

### Курсова работа по Операционни системи

Този проект представлява съвременно, трислойно микроуслужно уеб приложение за генериране на случайни вдъхновяващи мисли. Проектът е напълно контейнеризиран чрез **Docker** и се управлява централизирано с **Docker Compose**.

---

## 1. Структура на проекта

```text
quotes-project/
├── backend/
│   ├── app.py                  # Код на Flask API-то (Логика и връзка с Redis)
│   ├── Dockerfile              # Docker конфигурация за бекенда
│   └── requirements.txt        # Зависимости (Flask, Redis, Flask-CORS)
├── frontend/
│   ├── index.html              # UI интерфейс и JavaScript логика
│   ├── nginx.conf              # Настройки на Nginx (Reverse Proxy)
│   └── Dockerfile              # Docker конфигурация за фронтенда
└── compose.yml                 # Docker Compose файл за оркестрация на услугите
```

---

## 2. Как работят компонентите?

Приложението е разделено на **3 независими микроуслуги**, всяка работеща в собствен изолиран контейнер:

- **Frontend (Nginx):** Уеб сървър, който сервира потребителския интерфейс (HTML/CSS) на порт `8081`. Той действа и като Reverse Proxy, пренасочвайки заявките от браузъра (`/api/quote`) навътре към бекенда.

- **Backend (Flask):** Python уеб сървър, който съдържа логиката на приложението. При всяка заявка той избира напълно случаен цитат от база данни с 15 уникални мисли, за да поддържа динамичност.

- **Cache/Database (Redis):** Бърза in-memory база данни. При всяко кликване на бутона от потребителя, Flask извиква атомарната команда `incr()`, която увеличава глобалния брояч на кликванията в паметта на Redis. Тъй като данните се пазят в контейнера, броят на кликванията **не се нулира** при обновяване (Refresh) на страницата.

---

## 3. Комуникация между услугите

Всички контейнери са свързани в обща изолирана виртуална мрежа, автоматично създадена от Docker Compose. Комуникацията се осъществява чрез вградената Docker DNS система посредством имената на самите услуги:

1. Потребителят натиска бутона → Браузърът изпраща заявка към Nginx (frontend).
2. Nginx пренасочва заявката към Flask сървъра на адрес `http://backend:5000/`.
3. Flask се свързва с базата данни на адрес `redis-cache:6379`, увеличава брояча и връща финалния резултат обратно към потребителя.

## 4. Инструкции за изграждане и стартиране

Приложението е конфигурирано да се вдигне с **една-единствена команда**, без нужда от предварително инсталиране на Python, Nginx или Redis на хост машината.

### Стъпки за стартиране

1. Отворете терминала в главната папка на проекта (`quotes-project/`).

2. Изпълнете следната команда за автоматично изграждане на образите и стартиране на контейнерите:

```bash
docker compose up --build
```

3. След като контейнерите заредят, отворете уеб браузър на адрес:

```
http://localhost:8081
```

### Спиране на приложението

За безопасно спиране на услугите и почистване на контейнерите от паметта, натиснете `Ctrl + C` в терминала и изпълнете:

```bash
docker compose down
```
