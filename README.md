# Генератор на позитивни цитати (quotes-generator)
### 🎓 Курсова работа по Операционни системи

---

## 📝 Описание на проекта

Този проект представлява съвременно, трислойно микроуслужно уеб приложение за генериране на случайни вдъхновяващи мисли. Проектът е напълно контейнеризиран чрез **Docker** и се управлява централизирано с **Docker Compose**. Приложението демонстрира работа с уеб сървър, RESTful API и *in-memory* база данни (кеш).

---

## 🏗️ Архитектура и Микроуслуги

Приложението е разделено на **3 независими микроуслуги**, всяка работеща в собствен изолиран контейнер:

```text
  [ Потребител ]
        │
        ▼ (Порт 8081)
┌─────────────────────────────────┐
│       frontend (Nginx)          │  <-- Сервира HTML/CSS и действа като Reverse Proxy
└─────────────────────────────────┘
        │
        ▼ (Вътрешна Docker мрежа)
┌─────────────────────────────────┐
│       backend (Flask)           │  <-- Избира случаен цитат и обработва логиката
└─────────────────────────────────┘
        │
        ▼ (Вътрешна Docker мрежа)
┌─────────────────────────────────┐
│     redis-cache (Redis)         │  <-- Пази и инкрементира глобалния брояч
└─────────────────────────────────┘
```

### 🔹 Компоненти

- **Frontend (Nginx):** Уеб сървър, който сервира потребителския интерфейс (HTML/CSS) на порт `8081`. Той действа и като Reverse Proxy, пренасочвайки заявките от браузъра (`/api/quote`) навътре към вътрешната Docker мрежа.

- **Backend (Flask):** Python уеб сървър, съдържащ REST API логиката. При всяка заявка той избира напълно случаен цитат от база данни с 15 уникални мисли, за да поддържа висока динамичност.

- **Cache/Database (Redis):** Бърза in-memory база данни. При всяко кликване на бутона, Flask извиква атомарната команда `incr()`, която увеличава глобалния брояч на кликванията в паметта на Redis. Тъй като данните се пазят в контейнера, броят на кликванията е защитен от презареждане на страницата.

## 🌐 Комуникация между услугите

Всички контейнери са свързани в обща изолирана виртуална мрежа, автоматично създадена от Docker Compose. Комуникацията се осъществява чрез вградената Docker DNS система посредством имената на самите услуги:

1. Потребителят натиска бутона → Браузърът изпраща заявка към Nginx (frontend).
2. Nginx проксира заявката към Flask сървъра на адрес `http://backend:5000/`.
3. Flask се свързва с базата данни на адрес `redis-cache:6379`, увеличава брояча в паметта и връща финалния JSON резултат обратно към уеб интерфейса.

## 🚀 Инструкции за изграждане и стартиране

Приложението е конфигурирано да се вдигне с **една-единствена команда**, без нужда от предварително инсталиране на софтуер (Python, Nginx или Redis) на хост машината.

### 🛠️ Стъпки за стартиране

1. Отворете терминала в главната папка на проекта (`quotes-project/`).

2. Изпълнете следната команда за автоматично изтегляне, изграждане на образите и стартиране на контейнерите:

```bash
docker compose up --build
```

3. След като контейнерите заредят успешно, отворете уеб браузър на адрес:

```
http://localhost:8081
```

### 🛑 Спиране на приложението

За безопасно спиране на всички услуги и изчистване на контейнерите от системната памет, натиснете `Ctrl + C` в терминала и изпълнете:

```bash
docker compose down
```
